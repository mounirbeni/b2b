# MBN Health — نظرة عامة على المشروع

> ملف تعريفي بالمشروع: الهدف منه، لمن هو موجَّه، وكيف تنتظم أجزاؤه.
> (الوثائق بالعربية، والمصطلحات التقنية بالإنجليزية كما هي في الكود.)

---

## 1. ما هو المشروع؟

**MBN Health** منصّة صحّية من طرفين (two-sided platform):

1. **سوق حجز للمرضى (Marketplace):** يبحث المريض عن عيادة حسب التخصّص والمدينة، ويحجز موعده إلكترونيًا مباشرة — بلا مكالمات ولا انتظار.
2. **نظام إدارة للعيادات (Clinic OS):** تحصل كل عيادة على لوحة تحكّم كاملة لإدارة المواعيد، الطابور، والمرضى، واستقبال الحجوزات القادمة من المرضى فورًا.

يشرف على المنصّة **مشرف (Admin)** يوافق على العيادات ويدير اشتراكاتها.

### الهدف
- **للمريض:** حجز طبّي موثوق وسريع (أقل من دقيقتين)، مع تذكير تلقائي بالموعد.
- **للعيادة:** استبدال دفاتر المواعيد الورقية بنظام رقمي لحظي يقلّل الغياب ويرتّب العمل.
- **للمنصّة:** الربط بين الطرفين مع ضمان الجودة (عيادات معتمدة فقط تظهر للعموم).

## 2. المستخدمون والأدوار (Roles)

| الدور | كيف يدخل | ماذا يفعل | مساره |
|------|-----------|-----------|--------|
| **مريض (patient)** | يسجّل حسابًا في `/patient/register` | يبحث، يحجز، يتابع حجوزاته | `/patient/*`, `/search`, `/clinics/[slug]` |
| **عيادة (clinic)** | تسجّل في `/register` | تدير المواعيد والمرضى والاستقبال | `/medflow/*`, `/settings` |
| **مشرف (admin)** | بريده ضمن `ADMIN_EMAILS` | يوافق/يعلّق العيادات، يدير الاشتراكات | `/admin` |

> الدور يُحفظ داخل جلسة المصادقة (JWT)، و`middleware.ts` هو الذي يوجّه كل دور إلى منطقته ويمنعه من مناطق الآخرين.

### رحلة كاملة (مثال)
1. عيادة تسجّل ← حالتها `PENDING` وتعمل داخليًا فقط.
2. المشرف يوافق من `/admin` ← تصبح `APPROVED` وتظهر في الدليل العام.
3. مريض يسجّل، يبحث، يجد العيادة، ويحجز موعدًا.
4. الموعد يظهر **فورًا** في لوحة تحكّم العيادة (`/medflow/reception`).
5. العيادة تدير الموعد يوم الزيارة (بدء الاستشارة، إنهاء…).

## 3. البنية التقنية (Tech Stack)

| الطبقة | التقنية |
|--------|----------|
| الإطار | **Next.js 14** (App Router, Server Components) |
| اللغة | **TypeScript** |
| قاعدة البيانات | **PostgreSQL** (مستضافة على **Neon**) عبر **Prisma ORM** |
| المصادقة | **NextAuth (Auth.js v5)** — استراتيجية JWT، مزوّد Credentials |
| الواجهة | **Tailwind CSS** + مكوّنات مبنية على **Radix UI** (نمط shadcn/ui) |
| الأيقونات | **lucide-react** |
| الرسائل | **Twilio** (تذكير عبر WhatsApp / SMS) |
| الاختبارات | **Playwright** (اختبارات e2e) |
| الاستضافة | **Vercel** |

## 5. نماذج البيانات (Database Models)

المصدر: `prisma/schema.prisma`

| النموذج | الوصف |
|---------|--------|
| **User** | حساب مالك العيادة (أو المشرف). |
| **Clinic** | العيادة (tenant). تملك المرضى والمواعيد. لها `status`: `PENDING / APPROVED / REJECTED / SUSPENDED`. |
| **Subscription** | اشتراك العيادة (خطّة + حالة)، يُدار يدويًا من المشرف. |
| **Patient** | سجلّ مريض **داخل عيادة معيّنة** (منفصل عن حساب المريض). |
| **Appointment** | موعد: تاريخ، مدّة، حالة، نوع، مربوط بمريض وعيادة (وربّما بحساب مريض). |
| **PatientAccount** | حساب دخول المريض (منفصل، يسمح بالحجز عبر عدّة عيادات). |
| **ReminderLog** | سجلّ رسائل التذكير المُرسلة (WhatsApp/SMS) وحالتها. |
| **Account / Session** | جداول NextAuth القياسية. |

> **عزل البيانات (Multi-tenancy):** كل استعلام في التطبيق مُقيّد بـ `clinicId`، فلا ترى عيادةٌ بيانات عيادة أخرى إطلاقًا.

### حالات الموعد (Appointment status)
`SCHEDULED → CONFIRMED → WAITING → IN_PROGRESS → COMPLETED` (أو `NO_SHOW` / `CANCELLED`).

## 6. حالة كل جزء (ما هو حقيقي مقابل معاينة)

النظام مبني على قاعدة بيانات حقيقية، لكن ليست كل الوحدات مربوطة بجداول بعد:

**✅ مربوطة ببيانات حقيقية:**
- الصفحة الرئيسية، البحث، صفحة العيادة، والحجز العام.
- تسجيل الدخول/الحساب لكل الأدوار.
- **MedFlow:** لوحة التحكم، المواعيد، الاستقبال، المرضى، التقارير، التقويم.
- لوحة المشرف، وتذكيرات WhatsApp.

**🔵 معاينة تصميم فقط (Preview — تحمل شارة واضحة):**
- الأطباء، الفوترة، المختبر، الأشعة، المخزون، الموظّفون، الرسائل، الاستشارات (SOAP).
- السبب: لا توجد لها جداول في `schema.prisma` بعد. تعرض بيانات نموذجية لتوضيح التصميم فقط.

## 7. تشغيل المشروع محليًا

```bash
npm install                     # تثبيت الحزم (يولّد Prisma client تلقائيًا)
# املأ .env.local بـ: DATABASE_URL, DATABASE_URL_UNPOOLED, AUTH_SECRET
npx prisma migrate deploy       # تطبيق ترحيلات قاعدة البيانات
npm run dev                     # تشغيل الخادم على http://localhost:3000
npm run test:e2e                # تشغيل اختبارات Playwright
```

### متغيّرات البيئة الأساسية
| المتغيّر | الغرض |
|----------|--------|
| `DATABASE_URL` | اتصال قاعدة البيانات المُجمَّع (pooled) — وقت التشغيل. |
| `DATABASE_URL_UNPOOLED` | اتصال مباشر — تحتاجه ترحيلات Prisma (`migrate deploy`). |
| `AUTH_SECRET` / `NEXTAUTH_SECRET` | سرّ توقيع الجلسات. |
| `ADMIN_EMAILS` | قائمة بريد المشرفين (مفصولة بفواصل). |
| `TWILIO_*` | مفاتيح Twilio لرسائل WhatsApp/SMS. |
